## **Data Dictionary: Binance Crypto Lakehouse**

This document provides a comprehensive reference for all data assets within the Binance Crypto Lakehouse. It defines the business meaning, technical types, and transformation logic for each column across the Medallion Architecture.

## **1\. Gold Layer: Analytics & Reporting**

The Gold layer contains the final, high-performance tables modeled for business intelligence and quantitative analysis.

### **1.1 Table: fact\_klines**

**Description:** High-granularity transactional fact table containing 1-minute OHLCV candles. **Grain:** 1 record per symbol\_id per timestamp\_key.

| Column Name | Data Type | Constraint | Definition / Business Logic |
| ----: | ----: | ----: | ----: |
| symbol\_id | Long | PK, FK | Deterministic surrogate key linked to dim\_symbol. |
| timestamp\_key | Timestamp | PK, FK | Start of the 1-minute interval (UTC). Linked to dim\_time. |
| open | Decimal(18,8) | Not Null | Price of the asset at the first trade of the minute. |
| high | Decimal(18,8) | Not Null | Highest price traded during the 1-minute interval. |
| low | Decimal(18,8) | Not Null | Lowest price traded during the 1-minute interval. |
| close | Decimal(18,8) | Not Null | Price of the asset at the final trade of the minute. |
| volume | Decimal(18,8) | Not Null | Total **Base Asset** traded (e.g., total BTC in BTCUSDT). |
| quote\_asset\_volume | Decimal(18,8) | Not Null | Total **Quote Asset** value (e.g., total USDT in BTCUSDT). |
| number\_of\_trades | Integer | Not Null | Total count of individual trade executions. |
| taker\_buy\_base\_vol | Decimal(18,8) | Not Null | Volume initiated by market buyers (Aggressor volume). |
| price\_range | Decimal(18,8) | Derived | high \- low (Measures absolute volatility). |
| price\_change\_pct | Decimal(18,8) | Derived | (close \- open) / open (Returns return for the minute). |
| buy\_sell\_ratio | Decimal(18,8) | Derived | taker\_buy\_base\_vol / (volume \- taker\_buy\_base\_vol). Returns NULL if sell volume is 0\. |

### **1.2 Table: agg\_hourly\_summary**

**Description:** Hourly aggregations of trading activity, emitted 5 minutes after the hour closes.

| Column Name | Data Type | Definition / Business Logic |
| ----: | ----: | ----: |
| hour\_start | Timestamp | The start of the 1-hour window (e.g., 2026-04-17 10:00:00). |
| vwap | Decimal(18,8) | **Volume Weighted Average Price**: sum(quote\_asset\_volume) / sum(volume). |
| amplitude\_pct | Decimal(18,8) | Intraday volatility: (max(high) \- min(low)) / first(open). |
| buy\_pressure\_ratio | Decimal(18,8) | Sentiment indicator: sum(taker\_buy\_base\_vol) / sum(volume). |

### **1.3 Table: dim\_symbol**

**Description:** Descriptive metadata for each trading pair.

| Column Name | Data Type | Definition |
| ----: | ----: | ----: |
| symbol\_id | Long | Surrogate key generated by abs(hash(symbol)). |
| symbol | String | The natural trading pair key (e.g., "BTCUSDT"). |
| base\_asset | String | The asset being bought/sold (e.g., "BTC"). |
| quote\_asset | String | The asset used for pricing (e.g., "USDT"). |
| tracking\_start\_ts | Timestamp | The system date when data collection began for this asset. |

## **2\. Silver Layer: Quality & Integrity**

The Silver layer is the validated source of truth where all data quality rules have been applied.

### **2.1 Table: slv\_clean\_klines**

**Description:** Validated 1-minute klines. Identical schema to fact\_klines but contains the symbol string instead of symbol\_id.

### **2.2 Table: slv\_quarantine\_klines**

**Description:** Dead Letter Queue (DLQ) containing records that failed DLT Expectations.

| Column Name | Data Type | Definition |
| ----: | ----: | ----: |
| quarantine\_ts | Timestamp | Timestamp when the record was rejected from the main pipeline. |
| failure\_reason | String | The specific rule violated (e.g., valid\_price, valid\_symbol). |
| raw\_payload | String | The original raw JSON string before processing. |

## **3\. Audit Metadata (Global)**

These columns are required across all layers to ensure lineage and auditability.

| Column Name | Data Type | Definition |
| ----: | ----: | ----: |
| ingestion\_timestamp | Timestamp | When the data entered the Bronze layer from the source. |
| source\_file | String | The specific file path in the Volume where data originated. |
| gold\_processed\_ts | Timestamp | (Gold only) The time the record was materialized in the Gold layer. |
